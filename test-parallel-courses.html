Test - Amélioration de l'affichage des cours parallèles

Problème résolu

Lorsque plusieurs cours sont programmés dans le même créneau horaire (cours parallèles), l'affichage était tronqué et difficile à lire.

❌ AVANT - Affichage tronqué (problématique)

TD1/TD2/TP1
Math/Physique/Informatique
Dupont/Martin/Bernard
101/201/Lab1

Problèmes :

  • Noms des salles tronqués ("101/201/Lab1" → "101/201/L...")
  • Types de cours illisibles ("TD1/TD2/TP1")
  • Noms d'enseignants coupés
  • Difficile de comprendre combien de cours sont en parallèle

✅ APRÈS - Affichage optimisé (solution)

3 cours
Math + Physique...
Dupont + Martin...
101 + 201...

Améliorations :

  • Indication claire du nombre de cours ("3 cours")
  • Séparateur "+" plus lisible que "/"
  • Troncature intelligente avec "..." pour indiquer qu'il y a plus
  • Limite à 2 éléments visibles + "..." si plus
  • Tooltips avec informations complètes au survol

Modifications techniques apportées

1. Fonction getCombinedCourseInfo améliorée

  • Changement des séparateurs "/" vers " + " pour plus de lisibilité
  • Ajout de métadonnées pour l'affichage optimisé (combinedCount, combinedTypes, etc.)
  • Utilisation du type principal pour la couleur de base

2. Composant CourseBadge optimisé

  • Détection des cours combinés via course.isCombined
  • Affichage "X cours" au lieu des types combinés
  • Limitation à 2 éléments + "..." pour éviter la troncature
  • Ajout d'attributs title pour les tooltips
  • Masquage du compteur de sessions pour les cours combinés

3. Corrections TypeScript

  • Mise à jour des comparaisons de types pour le nouveau système TD1/TD2/TP1/TP2
  • Ajout d'exclusions dans tsconfig.json pour les fichiers de sauvegarde
  • Corrections des assertions de type pour éviter les erreurs de compilation

Résultat

Les cours parallèles sont maintenant affichés de manière claire et lisible, avec une indication du nombre de cours et des informations tronquées intelligemment.
